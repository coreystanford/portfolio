This project uses three data sources from Vancouver Open Data. The data is combined to produce a map of parks in Vancouver with details about the park status, the features of the park (including washrooms!), and hooks into the Google Maps JavaScript API in order to provide directions to the location of the selected park.

The Google Maps API also allows for HTML5 location detection. If enabled, the user's location is automatically plotted as the starting point on the map when a park is selected. The individual can then choose the mode of transportation they want to use, and the map updates with a new route, and an estimated duration.

The search functionality of the site is achieved by combining AngularJS to watch the text field for changes, which executes an AJAX call 450ms after a change has been registered (via the $http service). This call is then processed by PHP to parse select properties of a JSON file generated from the open data. The longitude and latitude are returned and plotted on the map, and updated in the list, giving the user multiple interaction avenues. An additional AJAX request is sent when a park is selected, whereafter the search window adapts to include additional information about the selected park in a smooth horizontal transition achieved using ng-style attributes.

Key Challenge:

A key challenge during this project was getting the unique identification key for each park to pass nicely from the JSON files, through the JavaScript, to the HTML and PHP, and back again.
